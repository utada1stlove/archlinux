### **个人 LaTeX 笔记模板使用说明 (终极完整版)**

恭喜你完成了这份功能丰富、高度定制化的 LaTeX 模板。本说明是它的“官方”使用手册，旨在帮助你完全掌握并高效利用它的每一个功能。

#### **1. 模板与编译**

* **文档类型**: `ctexbook`，A4纸张, 12pt字号, 双面布局。这是一个为中文书籍排版优化的专业文档类。
* **编译引擎**: **必须使用 `XeLaTeX`**。
* **项目文件结构**:
    ```
    /你的项目文件夹
    |-- coco.tex            (主文件)
    |-- /figures/             (存放图片)
    `-- /BIBbase/
        `-- myreferences.bib  (参考文献数据库)
    ```
* **核心编译流程**:
    * **日常编译**: `XeLaTeX` -> `XeLaTeX`
    * **更新参考文献后**: `XeLaTeX` -> `Biber` -> `XeLaTeX` -> `XeLaTeX`

#### **2. 自定义核心功能**

这是你模板中最具特色的部分，是你为了满足个人需求而量身打造的工具。

##### **2.1 代码块环境**

你拥有两个代码展示环境，可根据需要选用：

1.  **带行号的代码块 (`latex` 环境)**
    * **用途**: 用于展示需要详细讲解、或需要按行引用的较长代码。
    * **用法**:
        ```latex
        \begin{latex}
        % 你的代码...
        \end{latex}
        ```

2.  **不带行号的代码块 (`latex-nonumber` 环境)**
    * **用途**: 用于展示简短的代码片段或命令，使版面更简洁。
    * **用法**:
        ```latex
        \begin{latex-nonumber}
        % 你的代码...
        \end{latex-nonumber}
        ```

##### **2.2 自定义文本命令**

1.  **重要内容高亮 (`\important{...}`)**
    * **效果**: 将文字设置为 **楷体、小四号字**，用于突出核心定义或关键句子。
    * **用法**: `\important{民法典是社会生活的百科全书。}`

2.  **特殊引用格式 (`\lcite{...}`)**
    * **效果**: 将文字用全角方头括号 `〔...〕` 包裹，适合特定的引用或术语注释。
    * **用法**: `这体现了法律的〔谦抑性〕原则。`

3.  **中文字体切换命令**
    * **背景**: 你通过 `\setCJKfamilyfont` 定义了几个中文字体族。
    * **用法**: 在需要的地方，使用花括号 `{}` 限定作用范围，然后切换字体。
        * `{\kaishu ...}`: 将括号内的文字设置为**楷体**。
        * `{\songti ...}`: 将括号内的文字设置为**宋体**。
        * `{\heiti ...}`: 将括号内的文字设置为**黑体**。
    * **示例**: `默认是宋体，{\kaishu 这句话是楷体。} 又回到了宋体。`

##### **2.3 参考文献系统 (`biblatex`)**

这是模板中最专业的功能之一，请遵循以下流程：
1.  **添加数据**: 在 `BIBbase/myreferences.bib` 文件中添加 BibTeX 条目。
2.  **正文引用**: 使用 `\cite{文献key}` 来引用。
3.  **生成列表**: 文末的 `\printbibliography` 会自动生成符合国标的列表。
4.  **编译**: 严格遵守 `XeLaTeX` -> `Biber` -> `XeLaTeX` -> `XeLaTeX` 的顺序。

#### **3. 通用排版工具箱**

你的模板加载了多个强大的宏包，它们为你提供了丰富的排版工具。

##### **3.1 文本装饰与布局**

1.  **下划线 (`\uline{...}`)**
    * **来源**: `ulem` 和 `xeCJKfntef` 宏包。
    * **功能**: 提供能够支持中文并且可以自动换行的下划线。
    * **用法**: `\uline{这段重要的文字需要划线强调。}`

2.  **多栏排版 (`multicol` 环境)**
    * **来源**: `multicol` 宏包。
    * **功能**: 将一部分文本分为多栏，与 `\twocolumn` 不同，它可以自由开始和结束，并且栏宽更均匀。
    * **用法**:
        ```latex
        \begin{multicols}{3} % 分成三栏
          这里是需要分栏的很多文字...
          \lipsum[1] % 使用 lipsum 宏包的假文填充
        \end{multicols}
        ```

3.  **行间距调整 (`setspace` 宏包)**
    * **功能**: 方便地调整文档的行距。
    * **用法**:
        * **全局设置**: 在导言区使用 `\onehalfspacing` (1.5倍行距) 或 `\doublespacing` (双倍行距)。
        * **局部设置**: 在正文中使用 `spacing` 环境。
            ```latex
            \begin{spacing}{2.0} % 临时开启双倍行距
              这里的段落都是双倍行距。
            \end{spacing}
            ```

##### **3.2 特殊内容插入**

1.  **插入图片 (`graphicx` 宏包)**
    * **前提**: 图片文件需放在 `figures/` 文件夹内。
    * **用法**:
        ```latex
        \begin{figure}[htbp]
          \centering
          \includegraphics[width=0.8\textwidth]{图片文件名.png}
          \caption{图片标题}
          \label{fig:some_image}
        \end{figure}
        ```

2.  **绘制图形 (`tikz` 宏包)**
    * **功能**: 你加载了 `mindmap` 和 `trees` 库，可以方便地绘制脑图和树状图。
    * **用法**:
        ```latex
        \begin{tikzpicture}
          % 在这里输入 TikZ 的绘图代码
          \draw (0,0) -- (1,1);
        \end{tikzpicture}
        ```

3.  **增强表格 (`array` 宏包)**
    * **功能**: 这个宏包在后台工作，它增强了标准 `tabular` 和 `array` 环境的功能，比如提供了新的列类型，让你能更方便地控制表格列的对齐和格式。你无需直接调用命令，它会自动生效。

#### **4. 全局格式化设定**

这部分是你模板“颜值”的决定因素，它们在后台默默工作。

* **页面布局 (`geometry` 宏包)**: 严格设定了页边距，确保了打印和阅读的舒适性。
* **页眉页脚 (`fancyhdr` 宏包)**:
    * **样式**: 自定义的 `fancy` 样式。
    * **内容**: 页码在页面外侧，**章标题和节/小节标题在页面内侧**。支持长标题自动换行，非常高级。
* **章节标题 (`titlesec` 宏包)**: `\section` 和 `\subsubsection` 的字体大小、加粗、间距都经过了精细调整。
* **列表样式 (`enumitem` 宏包)**: 定义了从 `1.` 到 `(a)` 等多达7级的丰富列表样式，写读书笔记时非常方便。
* **脚注样式**: 脚注引用标记为 `[1]` 样式，分隔线为短横线，样式更醒目。
* **超链接 (`hyperref` 宏包)**: 内部链接为紫色，URL为青色，使PDF文档具备交互性的同时保持美观。
* **日期 (`\today`)**: 日期命令被重新定义为了一个固定的日期 `二〇一七年五月二十日`。如果你想使用当天的日期，可以找到 `\renewcommand{\today}{...}` 这一行，将它注释掉或删除。
* **占位符文本 (`lipsum` 宏包)**:
    * **功能**: 提供 `\lipsum` 命令，可以生成无意义的“乱数假文”，用于测试页面布局和样式。
    * **注意**: 这只在排版测试时有用，**最终的正式文档中应该删除 `\usepackage{lipsum}` 和所有 `\lipsum` 命令**。
